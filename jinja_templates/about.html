This is an Amazon Web Services AMI that has been customized for Django and GIS.

It's a great starting point if you are creating a backend for a mobile web application that requires geospatial capabilities.

Take a snapshot and use it for your Geospatial projects! It's all open source.

  • Ubuntu 12.04 (LTS) 64-bit [ami-4438b474]
  • ec2-ami-tools (Amazon EC2 AMI tools)
  • ec2-api-tools (Amazon EC2 API tools)
  • nginx (Small, but very powerful and efficient web server and mail proxy)
  • postgis (Geographic objects support for PostgreSQL -- common files)
  • postgresql-9.1 (Object-relational SQL database, version 9.1 server)
  • python-virtualenv (Python virtual environment creator)
  • supervisor (A system for controlling process state)
  • virtualenvwrapper (Useful extensions to virtualenv)
  • django 1.4 (Python web application framework)
  • green unicorn (Python WSGI HTTP Server)
  • jinja2 (Improved templating engine for Django)
  • TeraWurfl (PHP-based mobile browser identification server)
  • mongodb-10gen (An object/document-oriented database)
  • php5-fpm (Server-side, HTML-embedded scripting language)
  • django-debug-toolbar (Django debugging utility)
  • django-extensions (Custom extensions for Django Framework)
  • git-core (Distributed version control system)

Configuration Notes

Nginx is used for serving static files, and as proxy to Django/GreenUnicorn.

Supervisor monitors the Green Unicorn server that runs Django.

TeraWurfl runs on port 8800, responding to mobile device identification requests.

  • You can access the TeraWurfl interface on that port with a web browser by opening that port in your security group.
  • TeraWurfl uses the Mongo database to store device profiles.
